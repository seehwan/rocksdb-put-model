🔍 레벨별 컴팩션 동작 분석

L2의 지배적 영향

  • I/O 비중: 45.2% (전체 I/O의 절반 가까이)
  • WAF: 22.6 (가장 높은 Write Amplification Factor)
  • 특징: Major bottleneck, High WAF
  • 결론: L2 컴팩션이 FillRandom 성능의 핵심 제약 요소

레벨별 효율성

레벨 I/O 비중 WAF 효율성 특성
L0 19.0% 0.0 1.0 Flush only, Low WAF
L1 11.8% 0.0 0.95 Low WA, Minimal overhead
L2 45.2% 22.6 0.30 Major bottleneck
L3 23.9% 0.9 0.80 Moderate activity

⚡ 장치 열화 분석

성능 저하 패턴

  • 쓰기 성능: 15.8% 저하 (1688.0 → 1421.0 MiB/s)
  • 읽기 성능: 2.0% 저하 (2368.0 → 2320.0 MiB/s)
  • 유효 성능: 3.7% 저하 (2257.0 → 2173.0 MiB/s)
  • 사용 시간: 36시간 연속 쓰기 작업

열화 원인

  1. SSD 마모: 연속적인 쓰기 작업으로 인한 플래시 메모리 열화
  2. 컨트롤러 최적화: 초기 최적화 후 성능 안정화
  3. 파일시스템 영향: F2FS의 가비지 컬렉션 오버헤드
  4. 온도 영향: 장시간 사용으로 인한 온도 상승

🔄 FillRandom 성능 진화 분석

FillRandom 역설

장치 성능이 15.8% 저하되었음에도 불구하고 FillRandom 성능이 8.9% 향상

컴팩션 적응

  • 효과: +5% 성능 향상
  • 메커니즘: L2 병목에 대한 시스템 최적화
  • 특징: 시간이 지남에 따라 점진적 개선

시스템 최적화

  • 효과: +2% 성능 향상
  • 메커니즘: OS 및 파일시스템 최적화
  • 특징: 자동적인 성능 튜닝

워크로드 적응

  • 효과: +3% 성능 향상
  • 메커니즘: RocksDB 내부 최적화
  • 특징: 워크로드 패턴 학습

  • 비선형 특성: 단순 선형 모델로는 부족

레벨별 컴팩션 모델링

  • L2 중심: 전체 성능의 45.2% 차지
  • WAF 고려: 22.6의 높은 Write Amplification
  • 가중 효율성: 레벨별 특성을 반영한 모델링

시간 의존적 성능 변화

  • 장치 열화: 실험 중간에도 지속적 발생
  • 성능 진화: 시스템 적응으로 인한 성능 향상
  • 복합 효과: 열화와 적응의 상호작용

🎯 모델 개선 방향

  • FillRandom 특화 모델: 13.8% 오차 개선
  • 시간 의존적 모델링: 실험 중간 성능 반영
  • 레벨별 세분화: L2 병목 명시적 모델링

중기 개선 (3-12개월)

  • 동적 적응 모델: 시스템 최적화 반영
  • 워크로드별 특화: FillRandom, Overwrite, MixGraph
  • 환경별 모델: 다양한 시스템 환경 고려

장기 개선 (1-5년)

  • 머신러닝 통합: 복잡한 패턴 학습
  • 실시간 적응: 운영 중 모델 업데이트
  • 범용 모델: 다른 LSM-tree 시스템 적용

💡 핵심 교훈

1. 단순성의 중요성

복잡한 모델(v4 enhanced: 8.3%)보다 단순한 모델(v4 original: 5.7%)이 더 정확할 수 있음

2. 워크로드별 특성

FillRandom은 다른 워크로드보다 예측하기 어려우며, 특화된 모델링이 필요

3. 시간 의존성

정적 모델의 한계를 인식하고, 시간에 따른 성능 변화를 반영해야 함

4. 시스템 적응성

장치 열화에도 불구하고 시스템이 자동으로 최적화하는 메커니즘 존재

연구 기여도

  • 이론적 기여: L2 컴팩션의 지배적 영향 발견
  • 방법론적 기여: 4D Grid Interpolation 기반 모델링
  • 실용적 기여: 5.7% 오차의 정확한 성능 예측
  • 실험적 기여: 36시간 대규모 실험을 통한 검증
