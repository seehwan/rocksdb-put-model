# Phase-A: FillRandom 특화 모델 개발

## 🎯 목표
FillRandom 워크로드의 13.8% 오차를 5% 이하로 개선

## 📊 실험 설계

### 1. FillRandom 특성 분석

#### 1.1 다양한 키 분포 실험
- **균등 분포**: 모든 키가 동일한 확률로 선택
- **지수 분포**: 최근 키가 더 높은 확률로 선택
- **지수분포**: 특정 패턴을 가진 키 분포

#### 1.2 다양한 값 크기 실험
- **1KB 값**: 작은 값 크기
- **10KB 값**: 중간 값 크기
- **100KB 값**: 큰 값 크기

#### 1.3 다양한 동시성 수준 실험
- **1 thread**: 단일 스레드
- **4 threads**: 낮은 동시성
- **16 threads**: 중간 동시성
- **64 threads**: 높은 동시성

### 2. 시간 의존적 성능 변화 측정

#### 2.1 측정 주기
- **1시간 간격**: 상세한 성능 변화 추적
- **총 24시간**: 충분한 데이터 수집

#### 2.2 측정 항목
- **처리량**: ops/sec, MiB/s
- **지연시간**: 평균, 95%, 99% 지연시간
- **리소스 사용량**: CPU, 메모리, I/O

### 3. L2 컴팩션 세밀 분석

#### 3.1 컴팩션 상태 추적
- **컴팩션 주기**: L2 컴팩션 발생 주기
- **컴팩션 패턴**: 컴팩션 동작 패턴
- **WAF 변화**: Write Amplification Factor 변화

#### 3.2 I/O 병목 지점 분석
- **I/O 분포**: 레벨별 I/O 사용량
- **병목 지점**: 주요 병목 지점 식별
- **최적화 기회**: 개선 가능한 영역
