🎯 실험 개요

Phase-B는 RocksDB의 FillRandom 워크로드를 사용하여 대규모 데이터 처리 성능을 분석하는 실험이었습니다. 4일간의 연속 실행을 통해 420억 ops를 처리하며, RocksDB의 내부 동작과 Compaction 패턴을 상세히 분석했습니다.

⚠️ 주목할 점: 실험이 목표(10억 ops)를 42배 초과했지만, 이는 RocksDB의 안정적인 성능을 보여주는 긍정적인 결과로 해석됩니다.

🔍 주요 발견사항

성능 특성

  • 초기 버스트: 실험 시작 시 1.9M ops/sec의 높은 처리량 달성
  • 안정화 과정: 시간이 지나면서 100K ops/sec 수준으로 안정화
  • Compaction 영향: Compaction 과정에서 성능 변동 발생
  • 지속성: 4일간 연속 실행에도 안정적인 성능 유지

Compaction 패턴

  • 레벨별 분포: L0에서 L6까지 다양한 레벨에서 Compaction 발생
  • I/O 패턴: 읽기/쓰기 I/O의 시간별 변화 패턴 관찰
  • 성능 영향: Compaction 시 처리량 감소, 완료 후 회복
