RocksDB Put-Rate Model 검증을 위한 체계적인 실험 계획

Phase-A: Device Envelope 측정 | Phase-B: RocksDB FillRandom 실험

🎯 검증 목표

1. 모델 정확도 검증

  • 예측 vs 실제: 모델 예측값과 실제 RocksDB 성능 비교
  • 오차율 측정: 정량적 오차 분석 및 개선점 도출
  • 민감도 분석: 파라미터 변화에 대한 모델 반응성

2. 물리적 정확성 검증

  • Device Envelope: 실제 장치 특성 반영 정확도
  • Closed Ledger: 회계 폐곡선 검증
  • 에너지 보존: 물리적 제약 조건 만족 여부

3. 실용성 검증

  • 다양한 워크로드: FillRandom 외 다양한 패턴 검증
  • 스케일링: 다양한 시스템 크기에서의 성능
  • 실시간 예측: 동적 환경에서의 모델 응답성

📋 검증 단계별 계획

Phase-A: Device Envelope 측정 (완료)

목적: fio 그리드 스윕을 통한 정교한 Device Envelope 모델 생성

  • 4D 그리드 스윕: ρr, iodepth, numjobs, bs
  • 180개 측정점으로 완전한 모델 구축
  • 물리적 제약 검증 및 선형보간 정확도 확인
  • 결과: enhanced_envelope_report.json 생성

Phase-B: RocksDB FillRandom 실험 (진행 중)

목적: 실제 RocksDB FillRandom 워크로드 성능 측정 및 LOG 분석

  • 1TB 데이터 생성 (1,000,000,000 키)
  • 32개 쓰레드로 병렬 실행
  • RocksDB LOG 파일 실시간 분석
  • 현재 상태: 54% 진행 (259GB/1TB)

Phase-C: 모델 검증 (예정)

목적: v4 모델과 실제 RocksDB 성능 비교 검증

  • LOG 파일 파싱 및 성능 메트릭 추출
  • 모델 예측값과 실제 측정값 비교
  • 오차율 계산 및 분석
  • 모델 개선점 도출

🔬 검증 방법론

1. 엄격한 분리 (Strict Separation)

캘리브레이션 데이터와 검증 데이터를 완전히 분리하여 순환 검증을 방지합니다.

2. 홀드아웃 검증 (Holdout Validation)

전체 데이터의 일부를 홀드아웃하여 모델의 일반화 성능을 평가합니다.

3. 교차 검증 (Cross Validation)

다양한 워크로드와 시스템 설정에서 모델의 일관성을 검증합니다.

4. 통계적 유의성 (Statistical Significance)

충분한 샘플 수와 반복 실험을 통한 통계적 유의성 확보

🎯 성공 기준

정확도 기준

  • 예측 오차: ≤ 10% (우수), ≤ 5% (매우 우수)
  • 상관계수: R² ≥ 0.90 (우수), R² ≥ 0.95 (매우 우수)
  • 일관성: 다양한 워크로드에서 일관된 성능

물리적 정확성

  • Closed Ledger: ±10% 이내 오차
  • 에너지 보존: 물리적 제약 조건 만족
  • Device Envelope: 실제 장치 특성 반영

실용성

  • 실시간 예측: 1초 이내 예측 완료
  • 메모리 효율: 2GB 이하 메모리 사용
  • 확장성: 다양한 시스템 크기 지원
